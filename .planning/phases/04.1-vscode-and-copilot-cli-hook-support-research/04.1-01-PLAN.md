# Phase 4.1 Plan: VS Code and Copilot CLI Hook Support Research

## Objective

Make VS Code Copilot and Copilot CLI hook support explicit, researched, configurable, and verified enough for the `0.1.0` release candidate.

## Implementation Steps

1. Update setup internals to model hook surfaces with `both`, `vscode`, and `copilot-cli`.
2. Generate CLI-compatible lower camel case hooks by default and VS Code PascalCase hooks on request.
3. Normalize hook logger fields for VS Code-compatible and Copilot CLI-native payload names.
4. Update CLI help and README examples so users can see both telemetry sources and both hook surfaces.
5. Update the manual Copilot CLI validation flow so hooks are enabled during the prompt run.
6. Add focused fixture tests for hook config and payload normalization.

## Verification

- `npm test`
- `npm run check`
- `npm run smoke`
- `npm run verify:package`
- Local manual Copilot CLI flow with a cheap mini model and hook collection validation.

## Exit Criteria

- The code and README no longer imply only one Copilot source is tracked.
- The research artifact explains why the hook config shape was chosen.
- Hook config generation and hook payload logging are covered by tests.
