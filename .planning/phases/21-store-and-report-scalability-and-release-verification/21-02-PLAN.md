<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Parsed telemetry batch -> SQLite identities | Repeated, colliding, or oversized identity sets must not corrupt counts or evidence links. |
| Stored evidence/manual rows -> report ranking | Deduplication order can silently hide manual assignments or double-count usage. |
| Benchmark fixture -> performance claim | Timing-only or shallow shape comparisons can conceal functional drift. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-21-06 | Tampering | Batch identity maps | high | mitigate | Test existing, new, repeated, ignored, and conflicting identities with exact run-result IDs and transaction rollback. |
| T-21-07 | Denial of Service | SQLite bind sets | medium | mitigate | Deduplicate inputs and chunk every `IN` query below the runtime bind limit; benchmark thousands of records. |
| T-21-08 | Tampering | Manual report dedupe order | medium | mitigate | Derive ranking assignments from raw manual rows before usage dedupe and deep-test multiple manual labels per usage. |
| T-21-09 | Information Disclosure | Benchmarks and fixtures | medium | mitigate | Generate synthetic metadata only, keep content capture disabled, and clean temporary stores in `finally`. |
| T-21-10 | Repudiation | Performance evidence | low | mitigate | Emit package/Node versions, fixture sizes, semantic-equivalence fields, and separate timing dimensions; fail before output on drift. |
</threat_model>
