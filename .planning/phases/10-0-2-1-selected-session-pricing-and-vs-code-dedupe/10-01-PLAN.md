<threat_model>
Trust boundaries:
- Local telemetry/session files are untrusted input and may contain sensitive prompt content.
- The SQLite store is local user data and must preserve audit evidence without leaking full prompts by default.
- Report output is user-facing and must not claim official billing authority.

STRIDE register:
- Tampering: malformed local session files can contain misleading IDs or prices. Mitigation: keep existing parser validation, warnings, and fixture tests for malformed inputs; do not trust cache metadata as numeric billing unless numeric tokens exist.
- Information disclosure: duplicate repair and alias diagnostics must not persist full prompts. Mitigation: only persist IDs, source/session metadata, labels, pricing fields, and redacted raw records according to existing privacy defaults.
- Repudiation: merging duplicates can obscure original evidence. Mitigation: preserve raw_records, pricing diagnostics, aliases, source fields, and relink label_evidence before deleting duplicate usage_records.
- Denial of service: broad refresh scans can feel hung. Mitigation: targeted changed-file behavior or explicit progress/diagnostic output.
</threat_model>
