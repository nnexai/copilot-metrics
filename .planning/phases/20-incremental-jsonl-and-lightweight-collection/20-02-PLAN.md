<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Copilot/VS Code hook stdin → hook-only process | Hook payloads can contain malformed JSON, sensitive prompts, paths, and attacker-controlled strings. |
| Setup path/command values → generated shell command | Local paths and package-cache locations cross into shell-executed hook configuration. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-20-05 | Information Disclosure | `bin/copilot-metrics-hook.js` / `hook-logger.js` | high | mitigate | Reuse the existing redaction path, keep prompt preview opt-in, and test that raw prompt content is absent from stored records. |
| T-20-06 | Tampering | Hook stdin JSON | medium | mitigate | Parse exactly one JSON value through the shared stream reader and fail without appending on invalid input. |
| T-20-07 | Elevation of Privilege | Generated hook shell command | medium | mitigate | Reuse shell quoting for metrics home, executable paths, and event names; test spaces and package-cache command shapes. |
| T-20-08 | Denial of Service | Repeated fresh hook processes | low | accept | A daemon is explicitly deferred; minimize loaded modules and preserve configured hook timeouts to bound each invocation. |
</threat_model>
