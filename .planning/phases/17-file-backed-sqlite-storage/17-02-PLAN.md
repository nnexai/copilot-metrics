<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI/import data -> SQLite | Labels, session IDs, paths, payload fields, diagnostics, and report filters are untrusted local input. |
| Storage module -> filesystem | The DB file and WAL/SHM sidecars contain local usage metadata and must remain under the central user-level store path. |
| npm dependency -> runtime | `better-sqlite3` native binding becomes runtime code for every CLI install. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-17-01 | Tampering | `src/sqlite-store.js` prepared statements | mitigate | Use `db.prepare(...).run/get/all` with bound parameters for labels, session IDs, source paths, fingerprints, diagnostics, and report filters; reject string interpolation for user-controlled values. |
| T-17-02 | Information Disclosure | `openStore`, WAL/SHM files | mitigate | Preserve central user-level `paths.usageDb`, create parent directories with `0700`, best-effort chmod DB files to `0600`, and let package checks forbid SQLite artifacts. |
| T-17-03 | Elevation of Privilege | `better-sqlite3@12.10.0` | mitigate | Pin exact version in dependency/lockfile and leave native install/package validation to Plan 17-03 before phase completion. |
| T-17-SC | Tampering | npm dependency changes | mitigate | Do not add packages outside the approved research audit; any new package requires package legitimacy audit before install. |
</threat_model>
