# Phase 2: OTel Ingestion, Normalization, and Cost Model - Context

**Gathered:** 2026-05-30
**Status:** Ready for planning

<domain>
## Phase Boundary

Parse local JSONL telemetry from VS Code Copilot, Copilot CLI, and hook logs; normalize billing-like LLM spans; store raw and normalized records in a local SQLite database using npm/Node-portable dependencies only; and estimate GitHub AI Credits from a versioned local pricing table without double-counting root agent spans.

</domain>

<decisions>
## Implementation Decisions

### Local Store
- **D-01:** Use SQLite for the local queryable store.
- **D-02:** Use a Node/npm dependency for SQLite portability. Do not assume the `sqlite3` executable or other system dependencies are installed.

### the agent's Discretion
The agent may choose schema, import command shape, parser heuristics, fixture format, and pricing table representation. Keep the implementation local-first and conservative: unknown models or missing data should produce warnings instead of false precision.

</decisions>
