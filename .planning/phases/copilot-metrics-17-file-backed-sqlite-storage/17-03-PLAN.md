| Packed CLI -> temp home | The installed binary creates DB, WAL, and SHM files outside the checkout. |
| Benchmark/temp files -> package verifier | Generated SQLite artifacts must stay out of package contents and git-tracked source. |

| T-17-01 | Tampering | CLI smoke import/label/report commands | mitigate | Native smoke uses normal CLI arguments and prepared-statement-backed store helpers from Plan 17-02; no SQL is assembled from user labels in scripts. |
| T-17-02 | Information Disclosure | Temp package smoke homes and WAL/SHM sidecars | mitigate | Use temp directories, clean them on success/failure where practical, and keep package verifier blocking SQLite artifacts. |
| T-17-03 | Elevation of Privilege | `better-sqlite3` install/load | mitigate | Validate pinned dependency through packed tarball install and native `require` in a neutral directory; fail the phase on native load/install errors. |
| T-17-SC | Tampering | npm package contents | mitigate | `npm pack --silent --json` verification inspects actual package file list and blocks accidental database artifacts. |
</threat_model>
