# Plan 16-01: Manual precedence reports and release readiness

<objective>
Make active manual session labels the highest-precedence attribution source in ranking and reports, preserve automatic evidence for audit/detail views, verify correction behavior, and prepare/publish `copilot-metrics@0.5.0`.
</objective>

<must_haves>
- D-01: Manual labels rank before automatic labels, alphabetically among themselves.
- D-02: Default reports remain non-overlapping; additional manual labels require `--top-k` or `--all-matches` for totals.
- D-03: Preserve `label-confidence:v1`.
- D-04: Automatic scores remain unchanged below manual ranks; manual labels get a score above the automatic range.
- D-05: `remove` and `clear` immediately remove manual precedence and stale default provenance.
- D-06: Final ranks continue after manual labels.
- D-07: Matching manual and automatic labels merge into one output entry while automatic evidence remains stored/tracked.
- D-08: Manual assignments can create report/ranking entries for known sessions without automatic label evidence.
- D-09: Manual assignment attributes every usage row in the session at output time without DB rewrites.
- D-10: Manually labeled sessions are not unattributed.
- D-11: Existing selected-credit/label-name sorting remains primary.
- D-12: `--top-k` uses final rank and can include multiple manual labels.
- D-13: Human `report labels` overview stays compact.
- D-14: JSON `report labels` overview stays compact and aligned with human overview.
- D-15: Session-detail manual provenance lives inside `confidence`.
- D-16: `--detail` includes synthetic manual evidence rows.
- D-17: JSON exposes manual `created_at` and `updated_at` in detail/confidence metadata.
- D-18: Displaced automatic evidence remains visible in detail and merged confidence evidence.
- D-19: Overridden automatic labels use the existing final-rank inclusion model.
- D-20: Do not add `automatic_rank`.
- D-21: `source_summary` includes `manual`.
- D-22: Tests prove `set` and `clear` leave no stale manual provenance in default outputs.
- D-23: Phase includes actual `copilot-metrics@0.5.0` publishing after verification.
- D-24: Release work includes metadata, README/changelog, tag/release workflow, and neutral-directory validation.
- D-25: Release validation includes local checks, workflow watch, npm version verification, and neutral-directory `npx`.
- D-26: Neutral-directory `npx` proves binary/diagnostics; empty telemetry is not failure.
- D-27: After publish, update planning artifacts and archive/complete v0.5.0.
</must_haves>

<threat_model>
Manual labels alter local attribution semantics and release output. Primary risks are losing automatic evidence, double-counting sessions by default, leaking noisy provenance into overview reports, stale manual attribution after correction, and publishing an unverified package. Mitigations: compute manual precedence at report/ranking time, keep top-label default inclusion, restrict provenance to detail/session-detail surfaces, add stale-provenance tests for `set` and `clear`, and require local plus published-package validation before milestone close.
</threat_model>
