# Plan 15-01: Manual label assignment CLI and storage

<objective>
Provide durable manual session label storage and a CLI correction surface for assigning, listing, replacing, and removing active manual labels by exact session ID.
</objective>

<threat_model>
Manual labels are local metadata, but the command mutates persisted attribution state. Primary risks are accidental mutation of unknown sessions, empty labels entering storage, and loss of automatic evidence. Mitigations: require exact known `session_id`, reject empty labels before storage, keep assignments in a separate table, and never delete or rewrite `label_evidence`.
</threat_model>

<tasks>

## Task 1: Add active manual assignment storage

**Files:** `src/sqlite-store.js`

**Action:**
- Create `manual_label_assignments` in `initStore()` with `session_id`, `label`, `created_at`, and `updated_at`.
- Add a unique index or primary key on `(session_id, label)`.
- Export helpers for:
  - checking exact session existence across usage, label evidence, and hook events,
  - listing manual labels for a session,
  - adding labels idempotently,
  - removing labels idempotently,
  - setting/replacing the complete active label list,
  - clearing all labels.
- Return post-operation state with `session_id`, sorted `manual_labels`, `operation`, and `changed`.

**Verify:**
- `node --check src/sqlite-store.js`
- Tests can query `manual_label_assignments` and observe active state.

## Task 2: Add `copilot-metrics label <session-id> ...`

**Files:** `src/cli.js`

**Action:**
- Add help text for:
  - `label <session-id> list`
  - `label <session-id> add <label...>`
  - `label <session-id> remove <label...>`
  - `label <session-id> set <label...>`
  - `label <session-id> clear`
- Route the top-level `label` command before `report`.
- Use existing store locking and `ensureDataDirs()`.
- Reject unknown sessions before mutation or listing.
- Canonicalize labels through `canonicalLabel()`, reject labels that are empty after trimming, and do not regex-validate them.
- Human output should be compact and include the session ID plus current manual labels.
- JSON output should return the structured post-operation state.

**Verify:**
- `node --check src/cli.js`
- CLI smoke commands against temp homes return expected JSON.

## Task 3: Add focused CLI tests

**Files:** `test/report.test.js`

**Action:**

- Assert label detail and unattributed report JSON include session IDs needed for the manual correction workflow.

**Verify:**
