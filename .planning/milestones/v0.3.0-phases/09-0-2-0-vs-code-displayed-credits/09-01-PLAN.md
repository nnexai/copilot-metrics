# Plan 09-01: Displayed-credit evidence, pricing precedence, and reports

## Goal

Implement VS Code displayed-credit parsing and pricing precedence for `copilot-metrics@0.2.0`, preserving displayed credits as local observed display evidence that wins over token estimates when actual charge fields are absent.

## Scope

- VS Code chat-session fallback parsing.
- Pricing classification, conflict diagnostics, and inferred cache-read diagnostics.
- SQLite schema migration, insert, duplicate merge, and cost repair.
- Label, model, repo, detail, and unattributed report JSON/human output.
- Fixture tests and release docs/version updates.

## Tasks

1. Parse displayed-credit evidence
   - Extract `result.details` and equivalent details fields from VS Code chat-session root/request/result/metadata records.
   - Support numeric `credit`/`credits` and `0x`.
   - Preserve display text without storing prompt/assistant content.

2. Extend pricing classification
   - Add displayed-credit fields and basis selection after actual evidence and before token estimates.
   - Keep comparable token estimates populated.
   - Add material delta diagnostics and bounded inferred cache-read calculations.

3. Persist and merge evidence
   - Add usage columns for displayed credits/text/source evidence and inferred cache reads.
   - Update inserts, duplicate usage merges, and repair updates so `--refresh` can upgrade older upper-bound rows idempotently.

4. Extend reports
   - Include displayed and inferred fields in JSON outputs for labels, details, models, repos, and unattributed records.
   - Add compact human basis markers for actual, displayed, estimated, upper, included/zero, and unknown.

5. Verify and release-prep
   - Add tests for numeric credits, `0x`, absent/non-credit details, actual-over-displayed precedence, displayed-over-upper precedence, inferred cache math, report JSON/human markers, and refresh merge.
   - Run `npm test`, `npm run check`, `npm run smoke`, `npm run verify:package`, and `npm run check:readme-version`.
   - Update README, CHANGELOG, RELEASE, and package version to `0.2.0`.

## Acceptance Criteria

- VS Code chat session details with `0.8 credits` import displayed credit fields and select `displayed_credit` when no actual charge exists.
- `0x` details import as displayed/included zero evidence.
- Actual charge evidence remains selected over displayed credits.
- Displayed credits upgrade cache-unknown upper-bound rows during `--refresh` without creating duplicates.
- Bounded inferred cache-read diagnostics are shown separately from observed `cache_read_tokens`.
- JSON and human reports expose the displayed-credit basis and diagnostics.
