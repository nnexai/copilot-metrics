# Plan 07-01: Session fallback discovery, import, attribution, diagnostics, and release

## Goal

Make fallback session logs a default, setup-once ingestion path for `copilot-metrics@0.1.8`, then verify that reports remain useful when hooks and OTel files are unavailable.

## Tasks

1. **Default fallback source model**
   - Extend `src/paths.js` with platform-aware VS Code stable and Insiders workspace/session defaults plus Copilot session-state defaults.
   - Update `src/setup.js` persisted config so built-in fallback entries are explicit and custom entries remain additive.
   - Add setup tests for default and upgraded config.

2. **Discovery and diagnostics**
   - Refactor `configuredSourceFiles()` to return importable source files plus diagnostics.
   - Accept configured fallback files/directories, default fallback directories, `.jsonl`, and `.json` where supported.
   - Diagnose missing paths, unreadable paths, unsupported formats, content-only session files, and tokenless sessions with stable warning codes.

3. **VS Code fallback parser**
   - Add parser support for supported VS Code session `.jsonl` and `.json` files.
   - Extract session/request/response identifiers, model, token categories, timestamps, repo/cwd/branch/task hints, and transient prompt candidates for label extraction.
   - Normalize token-bearing VS Code fallback records into `usage_records`; keep existing response-id label linking behavior for label-only chat files.
   - Preserve idempotence via raw fingerprints.

4. **Copilot CLI session fallback hardening**
   - Keep `session-state/*/events.jsonl` import working from `COPILOT_HOME` and default `~/.copilot`.
   - Ensure shutdown model metrics produce label evidence through configured extractors and diagnostics when sessions are tokenless.

5. **Report integration**
   - Include fallback diagnostics in every report command JSON and human output.
   - Keep OTel/hook diagnostics compatible with the new diagnostic list.
   - Ensure advisory estimate wording is present for fallback diagnostics.

6. **Docs and release readiness**
   - Update README/CHANGELOG/RELEASE for v0.1.8 fallback behavior and privacy limits.
   - Preserve existing version-sync guardrails and package verification behavior.

7. **Verification**
   - Add fixture tests for platform default discovery, custom additive paths, VS Code `.jsonl`/`.json` fallback parsing, Copilot session import, custom extractor usage, idempotence, and report diagnostics.
   - Run `npm test`, `npm run check`, `npm run smoke`, and `npm run verify:package`.

## Acceptance Checks

- Running `setup` once writes fallback source config for VS Code stable, VS Code Insiders, and Copilot CLI.
- Running `report labels --json` with only fallback fixture files imports usage once, then produces identical output on a second run.
- A custom extractor configured in `config.json` is invoked for fallback-derived source data and suppresses built-in Jira extraction.
- Human report output includes actionable fallback diagnostics without full prompt content.
- Package verification passes with the files needed for fallback ingestion included.
