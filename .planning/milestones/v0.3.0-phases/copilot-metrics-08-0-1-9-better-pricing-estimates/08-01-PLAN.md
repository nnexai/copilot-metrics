# Plan 08-01: Pricing evidence, estimate confidence, and report semantics

## Goal

Implement the v0.1.9 pricing evidence model so imports preserve trusted observed local charge signals such as Copilot CLI `totalNanoAiu`, session-local pricing metadata, cache-read availability, and diagnostic evidence separately from advisory token-price estimates. Reports must expose actual, estimated, and upper-bound values distinctly in JSON and compactly in human output.

## Scope

- Extend usage normalization for OTel, VS Code chat fallback, and Copilot CLI session-state records.
- Extend SQLite schema, insert, duplicate update, and repair paths.
- Extend pricing classification and session-local price handling.
- Extend label, model, repo, detail, and unattributed reports.
- Add fixtures and tests for observed charge evidence, cache-unknown upper bounds, session-local prices, duplicate evidence preservation, and redacted diagnostics.

## Tasks

1. Pricing classifier and metadata model
   - Add pricing helpers in `src/pricing.js` for session-local USD-per-million-token prices, static fallback prices, cache-read availability, high-confidence estimates, upper-bound estimates, unknown-price warnings, included/zero evidence, and conflict warnings.
   - Keep existing `estimated_usd` / `estimated_ai_credits` compatibility fields, but add fields for `actual_*`, `upper_bound_*`, `pricing_basis`, `estimate_confidence`, `cache_read_status`, `pricing_source`, and diagnostics.

2. Normalization and import evidence
   - In `src/otel.js`, capture Copilot CLI `totalNanoAiu`, request cost/count, premium request counters, token details, and pricing diagnostics from `session.shutdown`.
   - In `src/ingest.js`, parse VS Code/Insiders model metadata such as `inputCost`, `outputCost`, `cacheCost`, `billing.token_prices`, `multiplierNumeric`, `pricing: 0x`, `cacheKey`, `cacheType`, and context utilization diagnostics without storing full prompt/assistant content.
   - Ensure missing numeric cache-read fields stay `unknown` instead of silently becoming `0` for fallback rows that have prompt/output tokens.

3. Store and duplicate merge
   - Add migration columns to `usage_records` for actual charge evidence, upper-bound estimate fields, pricing basis/confidence/source, cache-read status, pricing metadata JSON, and diagnostics JSON.
   - Update insert statements and duplicate `usage_identity` handling so re-importing the same exchange preserves or upgrades the strongest pricing evidence.
   - Extend cost repair to refresh the new estimate fields for existing rows without destroying observed charge evidence.

4. Reports
   - Update `src/reports.js` queries for labels, label summaries, model breakdowns, detail rows, model reports, repo reports, and unattributed reports.
   - JSON output must include auditable actual/estimated/upper-bound fields, basis counts, cache-read status, confidence, source/session/model evidence, and diagnostics.
   - Human output should stay compact with basis/confidence markers and a clearer estimate note.

5. Verification and release artifacts
   - Add or update fixtures for Copilot CLI `totalNanoAiu`, VS Code session-local prices with missing cache-read counts, `cacheKey`/`cacheType`, `multiplierNumeric: 0`, context utilization, and auth-like redaction.
   - Update `test/ingest.test.js` and `test/report.test.js`.
   - Run `npm test`, `npm run check`, `npm run smoke`, and `npm run verify:package`.
   - Update README, CHANGELOG, RELEASE, and package version to `0.1.9`.

## Acceptance Criteria

- Copilot CLI session shutdown records with cache-read tokens and `totalNanoAiu` produce trusted observed local charge fields plus comparable high-confidence estimates.
- VS Code/Insiders rows with token counts and session-local price metadata but no numeric cache-read count produce upper-bound estimates and cache-unknown diagnostics.
- Complete token buckets produce high-confidence estimates from session-local prices when available or static prices otherwise.
- Mixed reports do not collapse upper-bound values into exact estimates or observed charges.
- Duplicate imports keep one usage row and preserve the strongest pricing evidence.
- Cache metadata, context utilization, quota SKU, and inclusion evidence are diagnostics, not numeric billable token buckets.
- Auth-like values are redacted before persistence and output.
