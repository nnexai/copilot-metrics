# Plan 19-01: Optimize Report Queries and Release v0.6.0

## Objective

Reuse label report evidence/ranking work inside a single CLI command, prove report equivalence, benchmark report latency, and ship `copilot-metrics@0.6.0`.

## Tasks

1. Add `createLabelReportContext(dbPath)` in `src/reports.js`.
2. Thread optional shared context through label overview, summary, model breakdown, detail, and session-detail functions.
3. Update `src/cli.js` to create one shared context per `report labels` or `report label` command.
4. Add a fixture test comparing optimized context calls to independent report calls.
5. Add `npm run benchmark:reports`.
6. Run focused tests and full local gates.
7. Bump to `0.6.0`, update changelog, and run release checklist.
8. Publish/release through the established GitHub/npm path and validate `npx` from outside the checkout.

## Verification

The phase is complete when local gates pass, benchmark evidence is recorded, the release is published, and external `npx` validation confirms the installed package runs outside the checkout.
