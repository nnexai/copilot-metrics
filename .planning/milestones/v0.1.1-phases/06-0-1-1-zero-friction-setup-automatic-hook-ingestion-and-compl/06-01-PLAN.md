# Phase 6 Plan: Setup Persistence, Automatic Import, and Complete Token Reports

## Objective

Implement the `0.1.1` patch behavior: setup-once local configuration, automatic idempotent JSONL imports before reports, complete token category reporting, and non-misleading hook-only label output.

## Tasks

1. Add idempotent raw-record tracking.
   - Extend store schema with source file and raw fingerprint metadata.
   - Add migration-safe schema updates for existing local stores.
   - Filter parsed records before normalization and insertion.

2. Add report auto-import.
   - Load configured default source paths from local config.
   - Import VS Code, Copilot CLI, and hook JSONL sources before report queries.
   - Treat missing files as empty and repeated report runs as no-ops for already imported records.

3. Improve setup and hook behavior.
   - Persist config during `setup all`.
   - Keep `init` behavior intact.
   - Generate hook commands that work with installed executables as well as checkout-local JS files.

4. Complete report token output.
   - Add cache read, cache creation, reasoning, usage-record count, and token-bearing status to label human output.
   - Add cache/reasoning fields to label detail rows.
   - Keep JSON output machine-readable and backward-compatible.

5. Update tests, docs, and release metadata.
   - Add fixture tests for auto-import idempotency and hook-only labels.
   - Add setup persistence and hook command tests.
   - Update README, CHANGELOG, and package version for `0.1.1`.

## Verification

- `npm test`
- `npm run check`
- `npm run smoke`
- `npm run verify:package`

## Acceptance Criteria

- A user can run setup/init once and then run report commands without separate import commands.
- Report auto-import is idempotent across repeated runs.
- Label reports expose input, output, cache read, cache creation, and reasoning token totals in human and JSON output.
- Hook-only labels are visible but clearly identified as hook-only / no token-bearing usage.
- Missing or empty local stores produce clear report output.
