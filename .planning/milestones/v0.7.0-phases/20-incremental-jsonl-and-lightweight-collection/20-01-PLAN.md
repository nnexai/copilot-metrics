  <files>src/jsonl.js, test/ingest.test.js</files>
  <behavior>
    - Appending complete newline-terminated records after a saved byte boundary returns only those records with absolute line numbers.
    - An unchanged file at a valid boundary returns no records and does not read or split the historical prefix.
    - A trailing unterminated fragment produces neither a record nor a malformed warning and leaves its start byte uncommitted for the next call.
    - A malformed newline-terminated record produces the existing malformed_jsonl warning at its absolute source line.
    - Reads beginning in the middle of a multibyte UTF-8 sequence or with incompatible range metadata are rejected for caller-directed reset rather than silently corrupting JSON.
  </behavior>
  <action>Extend `readJsonl(file, options)` with an internal synchronous range/chunk path that accepts a byte start plus the number of already completed lines, reads forward without loading the historical prefix, parses only newline-complete records, and returns records/warnings together with the safe next byte position, completed-line count, and current file observations needed by the caller. Preserve the existing no-options and `afterLine` behavior for current consumers. Choose helper and result field names consistently within this plan; keep memory bounded by processing chunks and carrying only the current incomplete line. Write the behavioral tests before implementation, including UTF-8 and CRLF boundaries so byte offsets cannot drift from decoded character counts.</action>

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-20-01 | Tampering | `readJsonl` / checkpoint reset logic | medium | mitigate | Validate size, stable identity, byte boundary, and compatible context before resume; reset to complete parsing on mismatch. |
| T-20-02 | Denial of Service | Historical JSONL prefix | medium | mitigate | Use bounded chunk reads from the validated byte offset and skip unchanged files before parsing. |
| T-20-03 | Information Disclosure | Incremental raw-record path | high | mitigate | Route resumed records through the existing redaction/content-capture-disabled path and assert sensitive content is not stored. |
| T-20-04 | Repudiation | Malformed/partial source records | low | mitigate | Preserve absolute line numbers and emit warnings only for complete malformed lines so diagnostics remain attributable. |
</threat_model>
