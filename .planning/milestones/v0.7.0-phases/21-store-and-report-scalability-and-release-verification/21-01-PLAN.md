<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Existing local SQLite file -> migration runner | A legacy, partially migrated, corrupt, or newer-version store must not be silently damaged. |
| Imported local telemetry -> repair scheduler | Malformed or duplicate rows can trigger expensive repair work or unsafe survivor selection. |
| SQL planner -> performance assertions | Small fixtures can produce misleading plans unless representative cardinality and exact production SQL shapes are used. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-21-01 | Tampering | Ordered migrations | high | mitigate | Wrap each version transition, cleanup, index creation, and version advancement in one transaction; test rollback and retry. |
| T-21-02 | Elevation of Privilege | Future-version store handling | medium | mitigate | Reject schema versions above `CURRENT_SCHEMA_VERSION` before any mutation and retain owner-only file permissions. |
| T-21-03 | Denial of Service | Repair scheduling | medium | mitigate | Persist successful repair versions and schedule scans only for stale markers or relevant mutations; test unchanged paths with scan instrumentation. |
| T-21-04 | Tampering | Duplicate repair survivors | medium | mitigate | Retain existing stronger-pricing merge, stable survivor ID, evidence reassignment, and uniqueness regression tests. |
| T-21-05 | Repudiation | Query-plan evidence | low | mitigate | Assert exact production query shapes, named indexes, representative cardinality, and absence of automatic/full-scan fallbacks. |
</threat_model>
