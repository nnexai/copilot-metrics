<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Repository files -> npm tarball | Development planning, tests, telemetry, and local databases must not cross into the published package. |
| GitHub release -> npm trusted publishing | Tag/commit/version mismatch or failed CI could publish the wrong artifact or leave release state ambiguous. |
| npm registry -> isolated user invocation | Local checkout resolution can mask missing files, native dependency failures, or broken bins. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |

| T-21-11 | Information Disclosure | npm package contents | high | mitigate | Run package-content and dry-pack gates and reject planning/runtime/test/telemetry/database artifacts before audit. |
| T-21-12 | Tampering | Version/tag/release commit | high | mitigate | Synchronize package/lock/README versions, audit a clean exact commit, and create `v0.7.0` only after the audit. |

</threat_model>
