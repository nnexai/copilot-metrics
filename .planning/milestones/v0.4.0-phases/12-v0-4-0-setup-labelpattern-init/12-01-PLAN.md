# Phase 12 Plan: v0.4.0 setup labelPattern init

## Goal

Let users configure internal extractor label patterns during setup/init without hand-editing config, while keeping existing alias compatibility and extractor semantics.

## Tasks

1. Update CLI flag parsing to preserve repeated `--label-patterns` values.
2. Pass configured label patterns through `init`, `setup`, and setup subcommands that initialize the central data directory.
3. Validate configured regexes before writing config and persist them under canonical `labelPatterns`.
4. Keep existing read compatibility for `labelPatterns`, `labelPattern`, and `labelRegex`, and keep JavaScript `labelExtractors` replacement-only.
5. Update README and CLI help to describe setup-time configuration and separate pattern matching from confidence ranking.
6. Add focused tests for one pattern, multiple patterns, invalid regex rejection, repeated flag parsing, unchanged defaults, and extractor consumption.
