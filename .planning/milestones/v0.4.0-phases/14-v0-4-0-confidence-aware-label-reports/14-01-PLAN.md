# Phase 14 Plan: v0.4.0 confidence-aware label reports

## Goal

Make label reports use confidence rankings by default and add explicit top-k/all-match and session-detail modes.

## Tasks

1. Change `report labels` aggregation to top-label semantics so each ranked session contributes to one label by default.
2. Change `report label <id>` default aggregation to rank-1 sessions only.
3. Add `--top-k <n>`, `--top-k all`, and `--all-matches` broad inclusion modes.
4. Add JSON inclusion metadata with mode, top-k value, and overlap flag.
5. Keep existing `--detail` as evidence rows filtered by selected inclusion mode.
6. Add `--session-detail` for one row per included session with aggregate tokens, costs, ranking, models, and compact evidence summary.
7. Add report tests for top-label overview, label default rank-1, top-k/all-matches, and session-detail output.
